H5表单
 lable 标签的 for 属性可以关联一个input元素的ID，从而达到选中lable而选中对应的input标签
 所有的表单元素可以不在表单form标签中，可以根据表单的form属性来关联表单form的ID，实现在一个页面任意布局
 action：表单数据提交的服务器地址
 method：get属性值获取表单数据（在url地址栏会出现查询字符串），post属性值用来提交数据
（1）文本输入类型：
  《input type="设置样式" value="值"》
  1.text：默认文本框
  2.email：电子邮件输入框（只能进行简单的 @ 查询，具体的还是要用正则去匹配）
  3.tel：电话号码输入框（不会进行过滤，与默认的文本框一样）
  4.URL：url地址输入框（能进行简单的httt:// 开头的简单的验证，具体的验证还是要使用正则开解决）
  5.search：搜索输入框（输入框后有一个删除的叉叉可以清除输入的内容）
  6.number：数字输入框
  max：设置数字输入框能够设置的最大数字
  min：设置数字输入框能够设置的最小数字
  step：设置数字输入框每次递增或递减的跨度，默认为 1
（2）文本数字选择类型：
  1.range：特定范围内的数值选择器，会出现一个类似于音量的进度条，可以使用number的max/min/step等属性设置值范围和跨度
  2.color：颜色选择器，选中的颜色是以16进制形式展示
  3.datetime-local：时间选择器。可以是time / date / week / month
（3）H5表单新增属性：
  1.placeholder：用于设置文本框中的提示信息
  2.autocomplete：是否保存输入的历史信息，默认是on，关闭为off
  3.autofocus="autofocus"：自动获取文本焦点
  4.required="required"：验证类：不能为空值。
  5.pattern = “正则表达式”：当表单提交时（自动包含了开始和结束正则），通过正则表达式验证输入内容的合法性，pattern 有安全隐患，重要的验证必须放在服务器端！！！
  6.formaction 在submit里定义提交地址。常见应用场景：保存草稿
  7.formnovalidate 不做表单验证。例如 input type="submit" formnovalidate="formnovalidate"
  8.list:配合datalist来实现数据过滤选单效果
  案例演示：
     input type=“text” value=“” list =“datalist的ID”
     《datalist id=“list”》
      《potion value = "值"》html《/option》
      《potion value = "值"》css《/option》
      《potion value = "值"》js《/option》
     《/datalist》
badInput 提供的输入值浏览器无法转换 patternMismatch 输入值不满足pattern正则，返回true rangeOverflow 输入值超过range最大值，返回true rangeUnderflow 输入指小于的range最小值，返回true stepMismatch 验证range 的当前值 是否符合min、max及step的规则，不符合返回true tooLong 如果输入值超出了maxlength属性指定的值，返回true tooShort 官网没定义。 typeMismatch 输入值与预期类型不匹配，返回true 。例如没有按email格式输入。 valueMissing 如果输入值为空，返回true 。例如非空栏位没有输入值。 valid 是否满足了所有的验证，以上有一项为ture,返回false
（4）统一设置属性：在form中设置
  1.novalidate：统一设置：不对输入进行验证（默认开启验证）
  2.autocomplete：统一设置：输入是否自动填充
（5）表单监听事件：
  1.oninput事件：实时监听用户输入数据
  2.onchange事件：当域的内容被修改的时候触发
  3.setCustomValidity()方法：修改默认的提示信息
